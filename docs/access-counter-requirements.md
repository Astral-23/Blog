# アクセスカウンター要件定義（home固定・総和のみ）

## 1. 背景
- homeページに、2000年代初頭の個人ブログで一般的だった「あなたは◯◯人目」形式のアクセスカウンターを設置したい。
- 表示だけではなく、読み込みごとにカウントアップし永続化されることを必須とする。

## 2. スコープ
- 対象ページ: `home` (`/`) のみ。
- カウンター単位: サイトhome用の単一キー（`home`）。
- カウント対象: homeページ表示時のアクセス（API経由で加算）。

## 3. 機能要件
1. homeページ表示時にアクセス数を1加算する。
2. 表示項目は累計アクセス数（`total`）のみとする。
3. カウンターキーは `home` 固定とする。
4. APIは以下を提供する。
   - `POST /api/access-counter` : 指定キーのカウントを1加算して最新値を返す
   - `GET /api/access-counter?key=...` : 指定キーの現在値を返す（加算なし）
5. キーが不正な場合は `400` を返す。
6. 呼び出しは既存埋め込みタグ `md-embed` を使い、`type="counter"` で有効化する。
   - 例: `あなたは<md-embed type="counter"></md-embed>人目のお客様です。`

## 4. 非機能要件
1. 永続化方式はサーバー上のJSONファイルとする（単一VPS運用を前提）。
2. 破損したJSONや未作成ファイルでも、サービス継続を優先して初期化できること。
3. 見た目はレトロ感を残しつつ、既存ブログテーマ（`--bg` / `--surface` / `--line` / `--accent`）と調和させる。
4. 表示桁数は7桁ゼロ埋めを採用する。
5. 実装は複雑にしすぎず、単純な加算と表示を優先する。

## 5. 運用要件
1. データ保存先は環境変数 `ACCESS_COUNTER_STORE_PATH` で上書き可能とする。
2. 未指定時の保存先は `content/.meta/access-counter.json` とする。
3. 本番運用では、デプロイで上書きされないよう `ACCESS_COUNTER_STORE_PATH=/opt/blog/shared/access-counter.json` のようなアプリ外永続パスを使用する。

## 6. 受け入れ基準
- homeページを1回開くと、`home` の `total` が1増える。
- 画面上に「あなたは XXXXXXX 人目のお客様です。」形式の表示が出る。
- APIが `400` / `200` を正しく返す。
